| AIR-002 | Layer-B LLM-as-judge (`PR-EVAL-JUDGE-v1`) | Self-grading bias: the judge model shares training lineage, style preferences, and failure modes with the models it scores, so it systematically rates fluent, confident, same-family output too highly. | An inflated trust signal on exactly the answers that most need scepticism; users over-trust a wrong answer. | The judge is advisory only and never enters the `TrustScore` arithmetic — the composite is Layer-A deterministic signals exclusively. Layer A carries an independent, non-LLM grounding signal (`citation_marker_grounding`). The judge model is configurable and pinned separately from the answering slots, so it can be chosen from a different family. | Judge neutrality set (AC-042); cross-family calibration against the R2-S4 golden set — not yet run. | Yes — enabling `QUORUM_EVAL_JUDGE_API_KEY` in any environment, and any future proposal to let a judge verdict affect the numeric score. | Engineering lead | Not measured. The bias magnitude cannot be stated before the S4 human-labeled golden set exists; no number is claimed. |
| AIR-003 | Layer-B LLM-as-judge (`PR-EVAL-JUDGE-v1`) | Prompt injection via provider prose: the judge is a second LLM reading text the user and retrieved web pages can influence, which can instruct it to return maximum scores or to declare citation support verified. | A fabricated-citation answer is scored as faithful and support-verified, flipping the honest `unverified` band into a false confidence figure. | Provider prose enters the judge prompt as delimited, explicitly untrusted evidence, never as instructions; strict-JSON output contract with no free-form command channel; a non-conforming response yields no verdict; the verdict cannot raise the numeric score; judge rationale is rendered through the same escaping path as other provider prose. Threat-model entries T-011 and AB-007. | Judge injection set (T-011, AB-007); OC-2 honesty set (AC-041). | Yes — same key-enable approval as AIR-002. | Engineering lead | Not measured. Injection-resistance rate requires an adversarial corpus that does not exist yet; the mechanism is documented and tested for the no-verdict path only. |
| AIR-004 | Layer-B LLM-as-judge (`PR-EVAL-JUDGE-v1`) | Judge hallucination: the judge invents a rationale, cites evidence it never saw, or emits well-formed JSON with numbers unrelated to the text it was given. | A plausible but fictitious quality justification shown to the user; downstream calibration built on noise. | Judge output is metadata, not a score input; the rationale is presented as a model opinion, never as a verified finding; malformed output yields no verdict rather than a coerced one; temperature 0 and a pinned model reduce (do not eliminate) variance; the `unverified` band is the default so a missing verdict is the safe state. | OC-2 honesty set (AC-041); judge-verdict contract tests (`tests/unit/test_evaluation_judge.py`). | Yes — same key-enable approval as AIR-002. | Engineering lead | Not measured. Judge hallucination rate is an S4 golden-set measurement; stub-derived numbers are explicitly ineligible. |
| AIR-005 | `TrustScore` composite weights and bands | Uncalibrated thresholds: the Layer-A component weights and the score-to-band cut points are chosen by design judgement, not derived from labeled data, so a band could mean something different from what a user reads into it. | Users act on a band that has no measured relationship to correctness; a guardrail value ships as an unmeasured guess. | Per FS-6 the whole scoring surface is **advisory until S4**. The numeric score is suppressed and served as `unverified` unless a real judge verified citation support, which in every current configuration it has not — so no uncalibrated number is served today. Per-component contributions are surfaced so the composite is inspectable rather than opaque. Weights and cut points are constants in one place, to be re-derived from the S4 golden set. | Trust-vs-correctness calibration error (`docs/metrics/quality-ledger.md`, S2 mechanism / S4 measured number). | Yes — promoting the score from advisory to a surfaced confidence figure, and any change to the weights or band cut points. | Engineering lead | Not measured. Calibration error is `—` in the quality ledger until the S4 human-labeled golden set exists. |
