# Discovery Open Questions

| ID | Question | Owner | Needed Before | Impact | Status |
|---|---|---|---|---|---|
| DOQ-001 | Are the selected default OpenRouter model IDs currently available, priced acceptably, and compatible with the desired workflow? | Engineering | Implementation planning | Determines model defaults, compatibility checks, and cost model. | Answered by product owner: OpenRouter model availability, pricing, and search support have been verified. |
| DOQ-002 | How will web search/source links be implemented: model-native capability, OpenRouter-supported tool behavior, or separate search provider? | Architecture | Architecture | Determines integration design and citation reliability. | Answered: use OpenRouter search first, then fallback to Tavily search or another free search option. |
| DOQ-003 | What exact warning copy and interaction should appear for high-stakes topics? | Product/content design | Requirements | Impacts safety UX and acceptance criteria. | Answered: use the warning copy in `docs/33-content-design.md`. |
| DOQ-004 | What exact warning copy should appear for sensitive/private data near query entry? | Product/privacy | Requirements | Impacts privacy UX and acceptance criteria. | Answered: use the warning copy in `docs/33-content-design.md`. |
| DOQ-005 | Will MVP require accounts, anonymous use, or user-provided OpenRouter/API keys? | Product/architecture | Architecture | Impacts privacy, abuse prevention, cost control, and deployment. | Answered: require accounts before running queries; keep app-owned OpenRouter/Tavily keys server-side for default usage; enforce per-account quotas/rate limits with only one active query at a time; allow optional BYO OpenRouter key for more usage. |
| DOQ-006 | What is the acceptable max latency and max cost per completed query for the two-round debate workflow? | Product/engineering | Non-functional requirements | Impacts orchestration, streaming, cancellation, and model defaults. | Answered: target P50 <= 45s, P95 <= 120s, hard timeout 180s; target average cost <= $0.05/query, acceptable max <= $0.15/query, require user confirmation or block above $0.25 estimated cost. |
