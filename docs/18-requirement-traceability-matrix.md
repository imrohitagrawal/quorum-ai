- **NFR-004 — the 180-second RUN-LEVEL deadline is UNENFORCED (RB-5 / D2).** No mechanism in `src/` terminates a stuck run at 180s. The only 180s behavioural budget is `DEBATE_HARD_TIMEOUT_MS = 180_000` (`src/product_app/debate.py:47`), which gates whether debate *round 2* runs (measured from round-1 start) and degrades that run to a partial result — it does NOT bound total run wall-clock. The debate budget is pinned at its exact boundary by `tests/unit/test_debate_orchestration.py::test_debate_hard_timeout_budget_is_180s_and_gates_round_two_at_the_boundary`; provider-fault degradation honesty is pinned by `tests/resilience/test_fault_injection_lane.py`. NFR-004's "≥95% terminal within 180s" acceptance target must therefore be read as **not yet covered by an enforced deadline** — building a run-level deadline is a product change and its own PR (do not read this row as coverage of it).
