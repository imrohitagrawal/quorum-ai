# Study Glossary

Status: Git draft only. Owner: documentation. Evidence: repository source-of-truth docs.

## Terms

**Acceptance criteria** - Specific conditions that must be true for a requirement to count as done.

**AI eval** - A repeatable check that scores AI behavior, such as citation coverage, high-stakes warning behavior, false-consensus prevention, or prompt-injection resistance.

**AI grounding** - Connecting AI output to visible sources or approved context so users can inspect why an answer was produced.

**BYO key** - A bring-your-own OpenRouter key provided by a user for that user's account only.

**Consensus** - The part of the answer where model outputs materially agree.

**Decision support** - Information that helps a human decide, without claiming to be professional advice or an automated decision.

**Disagreement** - A material conflict or difference between model outputs that should remain visible to the user.

**Feature flag** - A server-side switch used to enable, disable, or roll back behavior without redeploying code.

**Grounding contract** - The product rules for sources, citations, uncertainty, refusal/warning behavior, and fallback behavior.

**MVP** - The smallest product slice that creates useful value and teaches whether the direction is right.

**OpenRouter** - The planned model access provider for the four configurable model slots.

**Partial result** - A terminal response that returns useful available outputs while clearly listing failed or missing steps.

**Prompt injection** - An attempt by user text, retrieved web content, or model output to override system rules, reveal secrets, or change intended behavior.

**Source reference** - A visible link or citation associated with a model answer or synthesis claim.

**Traceability** - The ability to follow a requirement to acceptance criteria, tests, code, release evidence, and production signals.
