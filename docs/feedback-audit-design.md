# Feedback Audit — Design & Prompt Specification

## What this is

A nightly job that reads the application's own event trail (synthesis quality,
provider performance, cost guardrails, safety warnings, model-slot choices) and
produces a structured Markdown report with actionable findings.

The report is appended to `feedback/audit-YYYY-MM-DD.md` and served at
`GET /feedback/audit` in the app. The operator reads it, applies diffs they
agree with, and ships the rest as PRs.

## Why it exists

It demonstrates a portfolio-grade feedback loop:

1. The app records structured quality signals for every query run (already built).
2. A scheduled job reads those signals and calls an LLM to analyze them.
3. The LLM produces proposed code changes as unified diffs — changes to the
   app's own source code.
4. The human operator reviews, ships, or rejects each finding.

This is the "the system improves itself" story. It is differentiated from
"added Prometheus" because the improvement loop is closed end-to-end within the
project's own data and toolchain.

## Event recorders consumed

| Recorder | Module | Key signals |
|---|---|---|
| `synthesis_event_recorder` | `synthesis.py` | `citation_coverage_ratio`, `false_consensus_preserved`, `status`, `duration_ms`, `high_stakes_warning_required` |
| `provider_event_recorder` | `providers.py` | `provider_path` (live vs fallback vs simulation), `duration_ms`, `fallback_used`, `source_count`, `model_id` |
| `model_slot_event_recorder` | `model_slots.py` | `model_slots` tuple (slot_number, model_id, search) |
| `cost_event_recorder` | `costs.py` | `threshold_action`, `estimated_cost_usd`, `confirmed`, `event_type` |
| `warning_event_recorder` | `safety.py` | `warning_type`, `acknowledged`, `event_type` |
| `debate_event_recorder` | `debate.py` | `round_number`, `duration_ms`, `status`, `timed_out` |

## Analysis prompt

The prompt is sent to the configured audit model (Haiku 4.5 by default, same as
the debate model — cheap and fast). It receives:

1. Aggregated statistics (counts, averages, failure rates) computed from the
   event trail — NOT the raw events. The LLM sees numbers, not PII.
2. The current contents of `src/product_app/model_slots.py` (the `DEFAULT_MODEL_IDS`
   tuple and the `InMemoryModelSlotEventRecorder` class).
3. The current safety regex pattern from `src/product_app/safety.py`.

It produces findings in a structured JSON envelope (see schema below). Each
finding has:

- **category**: one of `model_slot`, `safety_regex`, `citation_threshold`,
  `cost_threshold`, `pipeline_timing`, `provider_fallback`, `prompt_quality`
- **severity**: `high` | `medium` | `low`
- **title**: short, specific
- **evidence**: the numbers that motivated the finding
- **recommendation**: what to change
- **proposed_diff**: a unified diff string targeting a specific file
- **confidence**: 0.0–1.0

## Finding categories

| Category | What it detects | Example |
|---|---|---|
| `model_slot` | A default model slot has high failure rate or low citation coverage | "Slot 2 (claude-3-haiku) failed 40% of calls in the last 7 days — consider swapping to claude-haiku-4.5" |
| `safety_regex` | A high-stakes query type is not caught by the current regex | "Queries about 'mortgage' or 'refinance' are high-stakes financial advice but the regex only matches 'investment'" |
| `citation_threshold` | Citation coverage is consistently below 80% | "Average citation coverage over 7 days: 42%. The 80% target is not being met for any model." |
| `cost_threshold` | Cost estimates are wrong relative to actual provider bills | (requires cost tracking, deferred to L5) |
| `pipeline_timing` | A pipeline stage is consistently slow | "Debate round 2 averages 12s, 4× round 1. Consider reducing DEBATE_HARD_TIMEOUT_MS or simplifying round 2 prompt." |
| `provider_fallback` | A model consistently falls back to local simulation | "Slot 1 (gpt-4o-mini) returned LOCAL_SIMULATION in 60% of calls — the :online suffix may be rejected by this model." |
| `prompt_quality` | Synthesis sections are consistently falling back to templates | "Consensus section used local heuristic in 35% of runs — the synthesis model (gpt-4o-mini) may be too small for the consensus prompt." |

## Report format

```markdown
# Quorum-AI Feedback Audit — YYYY-MM-DD

**Window**: last 24h (runs completed between ...)
**Total runs analyzed**: N
**Generated by**: feedback_audit.py v0.1
**Audit model**: anthropic/claude-haiku-4.5

## Summary

| Category | Findings | High | Medium | Low |
|---|---|---|---|---|
| model_slot | 2 | 0 | 1 | 1 |
| safety_regex | 1 | 0 | 1 | 0 |
| ... | ... | ... | ... | ... |

**Overall health**: GOOD | DEGRADED | POOR
**Action required**: YES | NO

## Findings

### Finding 1 — [category] [severity]

**Title**: ...
**Evidence**: ...
**Recommendation**: ...
**Confidence**: 0.XX

```diff
--- a/src/product_app/model_slots.py
+++ b/src/product_app/model_slots.py
...
```

### Finding 2 — ...

...

## Negative findings (confirmed NOT problems)

The audit also checks for common failure modes that are NOT present:

- Citation coverage is NOT degraded for slot 3 (gemini-2.0-flash-lite)
- Safety regex does NOT need updating for "AI therapy" queries
- ...

This reduces false positives for the operator reading the report.
```

## Running the audit

### As a Fly cron job

```bash
fly machine run --app quorum-ai --env AUDIT_MODEL_ID=anthropic/claude-haiku-4.5 \
  -- python -m product_app.feedback_audit
```

### As a GitHub Actions scheduled workflow

```yaml
on:
  schedule:
    - cron: '57 3 * * *'  # 03:57 UTC daily
```

### As a one-shot local run

```bash
uv run python -m product_app.feedback_audit --window-hours 24 --output-dir feedback/
```

## Free-tier constraints

- ****: the audit model call costs ~$0.001–$0.003 per run (Haiku 4.5).
  At one audit per day, this is ~$0.03–$0.09/month — well within free tier.
- **Storage**: each audit report is ~5–15 KB. A year of daily reports is ~5 MB.
  No external storage needed; reports live in the repo or a Fly volume.
- **Compute**: the audit job runs in ~5–15 seconds (one LLM call + file writes).
  No dedicated infrastructure needed.
