# CLAUDE.md — ORBI AI Product Operating Model

This repository uses ORBI / Orbisynth-AI for new AI product incubation and shared AI operating standards.

Before doing product, requirements, architecture, implementation, testing, release, runbook, or learning work:

1. Read `AGENTS.md`.
2. Use `.claude/skills/orbi-ai-operating-model/SKILL.md` when turning a product idea into requirements, PRD/SRS/ADR, Jira-ready issues, quality gates, release readiness, runbooks, or learning notes.
3. Use `.claude/agents/orbi-ai-governance-agent.md` as a governance reviewer before implementation or release decisions.
4. Do not modify, move, rename, archive, merge, duplicate, or absorb Aegis or CiteVyn assets.
5. Do not store secrets, credentials, API keys, tokens, private keys, confidential customer data, or sensitive business data.
6. Build in small vertical slices and keep all generated artifacts reviewable.

System of record:
- Confluence = why, what, standards, requirements, decisions, learning
- Jira = executable work, status, ownership, traceability
- GitHub/code repository = implementation
- ORBI = new AI product incubation and shared operating model
